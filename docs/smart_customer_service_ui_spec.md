# 智能客服商业控制台 UI 设计规格

## 设计定位

这套 UI 面向可商业落地的微信私域智能客服，不只是“自动回复窗口”，而是一个客服运营控制台。核心目标是：

- 秒回常见问题，降低人工重复劳动。
- 自动识别高意向客户，推动留资和转化。
- 允许人工随时接管关键会话，降低风险。
- 后期可以持续添加客服 skills，并能配置、审计、灰度上线。

高保真原型文件：`docs/smart_customer_service_ui.html`

## 信息架构

主导航建议分为 8 个模块：

1. 总览工作台：展示今日咨询、自动解决率、高意向线索、响应时间。
2. 会话接管：左侧优先队列，中间聊天窗口，右侧 AI 决策面板。
3. 客服 Skills：管理报价、知识库、线索转化、风控质检等能力。
4. 知识库：维护产品、价格、起订量、交期、物流、售后、案例。
5. 线索 CRM：沉淀公司名、联系人、电话、预算、数量、使用日期。
6. 转化分析：按来源、话术、skill、客服人员分析成交贡献。
7. 风控质检：敏感问题、投诉、退款、发票、错误回复审计。
8. 系统设置：微信连接、模型配置、提示词、安全阈值、人员权限。

## 关键页面设计

### 总览工作台

工作台要让老板或运营一眼看到系统有没有创造价值：

- 今日新增咨询
- 高意向客户数量
- 自动解决率
- 平均响应时间
- 人工待处理数量
- 今日需跟进客户

### 会话接管

会话页面采用三栏结构：

- 左栏：客户优先队列，按高意向、需人工、待留资、报价中排序。
- 中栏：聊天上下文，展示用户消息、AI 回复、系统识别事件。
- 右栏：AI 决策面板，展示线索评分、命中 skill、置信度、建议动作。

这比普通聊天 UI 更适合商业使用，因为客服不是只回复消息，还要判断客户价值、推进流程、规避风险。

### Skills 中心

每个客服 skill 建议具备统一结构：

- 名称：例如报价初筛 Skill。
- 触发条件：关键词、意图、客户阶段、上下文状态。
- 输入字段：数量、预算、城市、日期、产品方向等。
- 输出动作：回复话术、打标签、创建线索、转人工、提醒跟进。
- 失败兜底：低置信度时转人工或只给建议不自动发送。
- 质检指标：命中次数、采纳率、转化率、人工修正率。

首批建议上线 6 个 skills：

- 报价初筛 Skill
- 知识库问答 Skill
- 线索转化 Skill
- 案例推荐 Skill
- 交期物流 Skill
- 风控质检 Skill

## 视觉规范

- 风格：B2B SaaS 后台，克制、可信、信息密度高。
- 主色：深青绿色 `#0f766e`，用于主按钮和在线状态。
- 辅色：蓝色用于知识/系统能力，琥珀色用于待确认，橙红色用于风险。
- 背景：浅灰 `#f4f6f8`，面板白色，边框 `#d9e0e8`。
- 圆角：统一 8px，避免过度卡片化。
- 字体：Windows 端优先 `Microsoft YaHei UI`。

## 商业化落地原则

- 先规则，后 AI：价格、起订量、交期、售后等强业务问题优先走确定性话术。
- AI 只在不确定问题中补充表达，不能编造价格、库存、发货承诺。
- 每条自动回复保留命中依据：skill、知识条目、置信度、路由原因。
- 高意向客户必须进入 CRM，不应只停留在聊天记录里。
- 敏感问题必须人工兜底：投诉、退款、发票、付款异常、法律、差评。
- 后期添加 skills 时保持统一配置模型，避免每个能力单独写死。

## 系统设置中的 API 接口设置

API 接口设置应放在“系统设置”里面，作为系统设置的一个子页签或设置分组，而不是放成和“总览工作台、会话接管、客服 Skills”同级的一级模块。模型配置不能写死在代码里。建议支持以下供应商：

- GeekNow API：你的专属接口，建议放在供应商列表第一位，并支持设为默认主模型。
- OpenAI：高质量兜底模型。
- DeepSeek：高性价比客服回复模型。
- 通义千问 / 阿里云 DashScope：国内企业云部署友好。
- 智谱 GLM：中文客服和知识库问答备用通道。
- Kimi / Moonshot：长上下文资料分析、话术整理。
- 百度千帆 / 文心：企业云账号可选接入。
- 腾讯混元：腾讯生态和企业微信场景可选接入。
- 火山方舟 / 豆包：国内低延迟回复通道。
- 硅基流动 / SiliconFlow：开源模型聚合接口。
- Ollama / 本地模型：离线测试和私有化部署备用。

每个接口配置项建议统一字段：

- 供应商名称
- 启用状态：启用并设为默认、启用为备用、暂停。
- Base URL
- API Key，必须加密保存，界面只显示掩码。
- 默认模型名称
- Temperature
- 最大输出 tokens
- 超时时间
- 重试次数
- 接口用途：主客服回复、RAG 问答、话术改写、复杂问题兜底、仅人工建议。
- 请求头 / 额外参数 JSON

商业化建议：

- 支持“测试连接”“保存配置”“设为默认接口”“切换备用模型”。
- 支持主备路由：例如 GeekNow 为默认主模型，DeepSeek/OpenAI 作为备用，接口失败自动切换。
- 支持按 skill 指定模型：报价和风控走低温度模型，话术优化和案例推荐可使用更强模型。
- 记录每次调用的供应商、模型、耗时、token、错误码和触发 skill，便于后期成本核算与故障排查。
- 设置每日预算、调用频率、失败率阈值，超过阈值时自动降级或转人工。

## 借鉴主流智能客服后的增强能力

参考 Zendesk、Intercom、Freshdesk 等成熟客服产品后，建议把以下能力纳入你的商业版路线：

- 多渠道统一收件箱：微信、企微、网页、抖音、小红书、电话、邮件统一进队列，客服不用在多个工具之间切换。
- 工单 + SLA：复杂问题、投诉、售后、付款异常自动建工单，并按客户等级和处理时限分派。
- AI 辅助人工：人工接管时提供知识推荐、话术改写、会话总结、下一步动作和一键建线索。
- 知识库自动优化：从未命中问题、人工改写、高频咨询里发现知识缺口，提醒补充知识。
- 多语言与翻译：预留中英等多语言客服能力，为跨境电商或海外客户扩展留空间。
- 质检与评测：用真实问题集评测模型、skills 和知识库，统计准确率、人工修正率、SLA 达成率。
- 开放集成：预留 CRM、ERP、表格、MCP/插件和外部 API 工具接口，让客服 skills 能真正执行业务动作。

## 下一版优先优化

已确定优先落地三项产品化能力：

- 系统设置真实化：API 配置需要有测试结果、最后修改人、修改时间、配置版本、错误日志和一键回滚，方便商业环境排障。
- Skill 创建向导：新增 skill 必须走标准流程，包括触发条件、必填字段、知识来源、模型选择、输出动作、失败兜底和上线测试。
- CRM 销售阶段流转：线索应按新线索、已留资、已报价、方案确认、打样/生产、成交/流失推进，每个阶段要有负责人和下一步提醒。

第二轮优化补充：

- 顶部增加全局健康状态：展示通道在线状态、API 延迟和异常入口。
- 顶部增加角色视角：老板视角、客服主管、一线客服、运营配置对应不同默认页面，减少信息堆叠。
- 会话接管增加客户画像：客户价值、历史沟通、历史订单、负责人、相似成功案例和下一步提醒。
- 总览页增加按角色进入工作，让不同岗位只看自己最需要的内容。

第三轮优化补充：

- 总览页增加多渠道健康矩阵：微信、企微、网页、小红书、抖音、电话/邮件按通道展示状态。
- 总览页增加今日运营待办：SLA 超时、知识库异常、API 成本接近阈值等主动提醒。
- 系统设置增加成本预算与自动降级：按每日预算、预警阈值、降级阈值控制模型调用成本。
- 系统设置增加开放集成：预留 CRM、ERP、表格、Webhook、MCP/外部工具接口。
- 系统设置增加备份恢复：配置、知识库、skills、会话和线索数据需要定时备份和可回滚。
- CRM 增加成交与流失复盘：把成交原因和流失原因反哺到知识库与 skill 评测集中。

第四轮优化补充：

- 总览页增加上线检查清单：知识覆盖、高风险转人工、真实问题评测集是商业交付前门槛。
- Skills 页面增加灰度发布：测试环境、灰度范围、上线门槛，避免新 skill 直接影响真实客户。
- 转化分析增加话术 A/B 实验：比较不同报价和留资话术，以转化率自动推荐胜出版本。
- 系统设置增加隐私合规：手机号、地址、联系人等敏感信息脱敏，训练和评测数据需要授权。

第五轮优化补充：

- 顶部增加快捷命令入口和 `Ctrl/Command + K`，快速打开系统设置、创建 Skill、查看线索和转化分析。
- 顶部通知按钮改为通知抽屉，集中展示 SLA、API 成本、知识库缺口等主动提醒。
- 知识库页增加空状态与导入引导，新客户首次部署时能明确下一步。
- 总览页增加异常与降级预案，覆盖模型失败、微信离线、知识冲突等真实运营问题。
- 增加 Escape 关闭命令面板/通知抽屉，提升键盘可达性。

第六轮优化补充：

- 顶部搜索支持按关键词跳转页面，例如 API、线索、知识库、Skill、风控、成本。
- 当前页面会保存到浏览器本地，刷新后保留用户所在模块。
- API 供应商列表可点击选择，并同步更新右侧“当前编辑”的供应商名称。

第七轮优化补充：

- CRM 页面增加线索字段字典，明确数据库字段、来源和业务用途。
- 转化分析页面增加关键事件埋点，后续可统计渠道量、模型成本、人工接管、线索转化和知识缺口。
- 总览页增加开发落地清单，拆解数据层、服务层、界面层的优先实现范围。
- API 供应商选择改为数据属性联动，同步更新 Base URL 和默认模型，避免后续开发误绑字段。

第八轮优化补充：

- 顶部通知按钮显示未读数量，打开通知抽屉后清零。
- API 测试连接和设为默认接口后，会同步更新顶部健康状态。
- 总览页增加商用验收标准，明确自动回复、线索、接口、知识库四个必须达标项。

## GeekNow API 对接说明

你提供的 `ComfyUI_GeekNow_Image_API` 里包含图片、视频和 LLM 三类接口。智能客服文本回复应优先对接 LLM 节点里的 OpenAI 兼容聊天接口：

- Base URL：`https://api.geeknow.ai/v1`
- Chat endpoint：`/chat/completions`
- 鉴权：`Authorization: Bearer <API_KEY>`
- 请求格式：OpenAI chat completions
- 默认模型可先使用：`gpt-4o-mini`，后续可按 GeekNow 实际可用模型替换

图片生成和视频生成接口属于素材/案例生成能力，后续可以做成独立 skill，例如“生成礼盒方案图”“生成宣传视频”，但不应混入客服主回复模型路由。

## 与当前项目的衔接

当前项目已经有 `scripts/ui_app.py` 作为桌面控制台入口，建议后续把它从“启动/日志控制台”升级为：

- 左侧导航 + 多页面布局。
- 运行状态页保留现有启动、停止、日志能力。
- 新增会话队列、AI 决策、skills 管理和线索列表。
- `config/customer_knowledge.yaml` 可作为知识库页面的数据来源。
- `core/customer_agent.py` 的 route、topic、confidence、citations 可直接映射到右侧 AI 决策面板。
