# 多平台统一客服终端方案

## 目标

把微信、小红书、拼多多、淘宝/天猫、抖音/抖店、快手/快手小店等渠道，统一接入当前智能客服系统。

最终效果：

- 一个终端看所有平台消息。
- 一个知识库回答所有平台客户。
- 一个客户线索池统一筛选高价值客户。
- 一个出图任务流程承接所有平台需求。
- 每个平台只负责收消息和发消息，不重复写客服大脑。

## 核心架构

```text
平台消息入口
  ├─ 微信 Adapter
  ├─ 小红书 Adapter
  ├─ 拼多多 Adapter
  ├─ 淘宝/天猫 Adapter
  ├─ 抖音/抖店 Adapter
  └─ 快手 Adapter
        ↓
统一 ChannelHub
        ↓
消息标准化：platform / sender / content / order / product / raw
        ↓
当前智能客服大脑
        ├─ 意图识别
        ├─ 知识库检索
        ├─ 真人风格回复
        ├─ 高价值客户评分
        ├─ 人工交接
        └─ 出图提示词提取
        ↓
按来源平台发回客户
```

## 已完成的基础改造

当前已新增：

- `core/channel_adapter.py`
- `core/channel_registry.py`
- `config/settings.yaml` 的 `channels` 配置

微信仍是默认启用渠道。其他平台已经进入配置清单，但未假装接通。

## 统一消息格式

每个平台的 Adapter 都必须输出：

```json
{
  "platform": "douyin",
  "channel_id": "douyin",
  "channel_name": "抖音/抖店",
  "sender": "客户昵称或平台用户ID",
  "content": "客户消息内容",
  "raw": {}
}
```

后续可扩展：

```json
{
  "order_id": "平台订单号",
  "product_id": "商品ID",
  "product_title": "商品标题",
  "buyer_level": "会员等级",
  "scene": "售前/售后/直播间/私信/评论",
  "attachments": []
}
```

## 平台接入原则

1. 优先官方 API 或官方服务市场授权。

这样稳定、合规、可商用，也更容易拿到订单、商品、售后上下文。

2. 第二选择是合规第三方聚合客服服务。

如果平台官方接口不直接开放给普通商家，可以用已获平台授权的聚合客服服务，再由它转发消息给当前系统。

3. 不建议逆向 App、抓包私信、模拟非公开接口。

这类方案短期看似能用，长期容易封号、丢消息、违反平台规则，也不适合承载客户资料和订单信息。

## 各平台建议路径

| 平台 | 推荐接入方式 | 优先级 | 备注 |
| --- | --- | --- | --- |
| 微信 | 当前 PC 自动化；后续企业微信/公众号官方接口 | 已接入 | 当前先保留 |
| 小红书 | 商家客服/官方授权能力或聚合服务 | 高 | 私信和店铺咨询要分开评估 |
| 拼多多 | 拼多多开放平台/商家授权或聚合服务 | 高 | 适合先接订单咨询和售后 |
| 淘宝/天猫 | 阿里/淘宝开放平台、千牛服务能力或聚合服务 | 高 | 重点是千牛客服与订单上下文 |
| 抖音/抖店 | 抖音开放平台/抖店开放能力或聚合服务 | 高 | 区分私信、抖店客服、直播线索 |
| 快手/快手小店 | 快手电商开放平台/商家授权或聚合服务 | 中 | 区分直播间、小店订单、短视频咨询 |

## 实施顺序

第一步：先统一渠道层。

- 已开始完成。
- 当前客服大脑不再直接依赖微信。
- 微信作为第一个 Adapter。

第二步：做“统一消息收件箱”。

- 控制台显示所有平台消息。
- 支持按平台筛选。
- 高价值客户、人工交接、报价准备度统一统计。

第三步：优先接一个电商平台。

建议先选淘宝/千牛、拼多多或抖店其中一个，因为它们更容易产生订单和高价值采购线索。

第四步：接出图流程。

所有平台客户都进入同一条链路：

```text
客户需求 → 提取产品/风格/尺寸/数量/文案 → 生成提示词 → 出图软件 → 人工修改 → 交付客户
```

第五步：再接小红书、快手等内容/私信型平台。

这些平台客户更偏种草、咨询和引流，适合重点做线索筛选和私域转化。

## 当前不能直接承诺的事

- 不能保证每个平台都有公开客服消息 API。
- 不能保证普通账号能直接拿到私信接口。
- 不能承诺用非官方方式长期稳定。
- 不能在未拿到平台授权资料前臆造接口。

后续每接一个平台，都需要先确认：

- 是否有商家账号。
- 是否有开放平台应用。
- 是否有 AppKey / AppSecret / AccessToken。
- 是否允许接收客服消息。
- 是否允许自动回复。
- 是否能拿订单、商品、售后上下文。

## 建议优先级

优先做：

1. 淘宝/千牛或拼多多
2. 抖音/抖店
3. 小红书
4. 快手/快手小店
5. 微信升级到企业微信/公众号官方接口

原因：你的业务是礼盒定制和成交转化，优先接有订单、商品、交易场景的平台，价值更高。
